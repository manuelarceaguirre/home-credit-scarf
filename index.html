Executive Summary

We built a leakage-safe, split-first unsupervised segmentation pipeline on the Home Credit Default Risk dataset and compared a classical baseline (PCA + clustering) against a self-supervised contrastive encoder (SCARF). We observed that PCA retained stronger linear predictive signal for a downstream logistic probe (AUC 0.7449 vs 0.6426 for SCARF), but SCARF produced substantially tighter clustering geometry and stronger risk stratification for segmentation: test silhouette 0.3855 vs 0.1797, and IV gains at shared cluster counts for k ≥ 4. We interpreted this as a meaningful tradeoff between linear separability and geometry-oriented segment structure. We concluded that SCARF is valuable for borrower segmentation use cases where coherent latent islands and risk-layered cluster profiles matter most, while PCA remains a strong baseline when linear discriminative performance is the priority.

1. Introduction & Motivation

We framed this project around a practical gap in credit analytics: supervised default prediction and unsupervised borrower segmentation solve related but different business problems. In production credit operations, we rarely need only a binary score. We often need borrower archetypes that can support policy design, pricing, intervention sequencing, collection strategy, and portfolio stress testing. In other words, segmentation is not a side task; it is a decision interface.

We recognized that high-dimensional tabular credit data is noisy, partially missing, semantically mixed, and not always linearly structured. If we rely only on linear dimensionality reduction followed by centroid clustering, we may obtain broad and overlapping partitions that are mathematically valid but operationally weak. Borrowers can occupy non-linear manifolds where local neighborhoods matter more than global variance directions. That mismatch motivated us to test representation learning as a preprocessing stage for unsupervised clustering.

Our central hypothesis was that self-supervised contrastive learning can build a latent space with better local geometry than PCA in this context. We expected that if we train an encoder to keep semantically perturbed views of the same borrower close while pushing different borrowers apart, we may uncover cleaner latent islands. Those islands should translate into stronger silhouette behavior and more distinct risk strata across clusters, even if linear probe performance is not maximized.

Hypothesis: We expected SCARF embeddings to improve unsupervised segmentation quality (geometry + risk separation), while acknowledging that linear discriminative metrics could move in either direction.

2. Literature Review

SCARF (ICLR 2022)

We selected SCARF because it is one of the most direct and practical contrastive frameworks for tabular data. SCARF generates two views by feature corruption rather than image-style augmentations and optimizes an InfoNCE/NT-Xent objective so that positive pairs (same sample under corruption) align while other batch samples act as negatives. For our setting, this was a natural fit because our table has mixed data types, many missing values, and large row count, and we needed a method that is conceptually simple enough to implement from scratch in PyTorch.

ReConTab (arXiv 2023)

We treated ReConTab as evidence that contrastive objectives can be made robust for tabular domains through corruption-aware objectives and stronger augmentation design. It reinforced our belief that representation learning for tabular data is no longer niche and that non-linear latent spaces can be materially better than purely linear transforms in segmentation-oriented workflows.

TCCL (arXiv 2025)

We used TCCL as recent context for the direction of the field: contrastive tabular methods continue to push representation transfer, clustering quality, and downstream adaptation. Even without implementing TCCL in this project, we used it to situate our work in a modern sequence and to justify why we prioritized SSL architecture and corruption strategy instead of stopping at PCA.

Why We Chose SCARF Over Other Alternatives

  • We chose SCARF over VAEs because VAEs impose an explicit generative bottleneck and often require careful likelihood assumptions that may be brittle on heterogeneous credit tables.
  • We chose SCARF over deep clustering hybrids in this phase because we wanted a clean representation-learning stage before clustering, not a tightly coupled joint objective that is harder to diagnose.
  • We chose SCARF over graph-based methods because our borrower table lacked a native, trustworthy relational graph; constructing one would add major modeling assumptions and possible leakage pathways.
  • We chose SCARF over heavier transformer pipelines because we prioritized reproducibility and clear implementation logic under CPU constraints.

4. Methodology - Preprocessing Pipeline

Split-First Design (Leakage Prevention)

We separated TARGET and borrower ID immediately, then performed an 80/20 stratified split before any imputation, scaling, encoding, or dimensionality reduction. We fit every transform on train only and applied it to test. This design is essential in financial risk modeling because tiny leakage channels can significantly inflate evaluation confidence.

# Split immediately after separating TARGET and ID
X_train_raw_df, X_test_raw_df, target_train, target_test = train_test_split(
    X_raw_df, target_series, test_size=0.2, random_state=SEED, stratify=target_series
)

DAYS_EMPLOYED Sentinel Anomaly (365243)

We corrected a critical anomaly before numeric imputation: DAYS_EMPLOYED = 365243, a known sentinel placeholder for unknown employment duration in this dataset. This value appears in roughly 18% of rows in common Home Credit analyses, and if we leave it untouched it contaminates medians, expands variance unnaturally, and destabilizes standardization. StandardScaler would otherwise treat sentinel rows as extreme yet meaningful values, distorting distance geometry.

# Fix sentinel before imputation/scaling
if "DAYS_EMPLOYED" in X_train_proc.columns:
    X_train_proc.loc[X_train_proc["DAYS_EMPLOYED"] == 365243, "DAYS_EMPLOYED"] = np.nan
    X_test_proc.loc[X_test_proc["DAYS_EMPLOYED"] == 365243, "DAYS_EMPLOYED"] = np.nan

Numeric Imputation: Train Median

We imputed numeric columns with train medians, not means. Financial variables in this dataset are heavy-tailed and skewed (credit amounts, annuities, incomes), and medians are more robust to outliers. We also persisted imputation dictionaries for inference consistency.

Missingness Indicators (>5%)

We created _MISSING binary flags for columns whose train missing ratio exceeded 5%. We did this for two reasons: (1) missingness can encode process information (documentation quality, underwriting friction, informal labor patterns), and (2) pure imputation can erase useful absence patterns. With 57 such indicators, we preserved this signal explicitly.

Categorical Handling: Ordinal vs Nominal

We separated ordinal and nominal categories. For NAME_EDUCATION_TYPE, we defined an explicit order from lower secondary to academic degree, mapped to prefixed ordered strings, and applied LabelEncoder. For nominal columns, we used pd.Categorical with train categories and then one-hot encoded via get_dummies(drop_first=True).

Train/Test One-Hot Alignment

We enforced category alignment using train-fitted category sets before dummy expansion, and then we reindexed test columns to train schema. This guaranteed identical dummy bases and identical dropped reference categories across splits.

for col in nominal_cols:

StandardScaler Fit on Train Only

We standardized only after all imputation and encoding. We fit StandardScaler on train and transformed both splits. Final shapes were (246008, 282) train and (61503, 282) test with zero missing residuals.

Bundling _MISSING Indicators with Parent Features for SCARF Corruption

A key methodological choice was grouping missingness flags with their parent variables in corruption logic. If we corrupt a parent without its missing flag (or vice versa), we can create semantically inconsistent synthetic views. We therefore bundled parent/indicator indices into shared corruption groups and treated one-hot category blocks atomically.

5. Methodology - Baseline (PCA + Classical Clustering)

Our baseline pipeline used train-fit PCA, K-Means model selection, Ward hierarchical clustering, and DBSCAN density clustering, all under leakage-safe split discipline.

PCA at 90% Variance

We fit PCA on train only, computed cumulative explained variance, and selected components to exceed 90% train variance. This required 146 components, yielding train shape (246008, 146) and test shape (61503, 146).

K-Means Selection (k=2..10)

We evaluated silhouette on train for k = 2..10, chose k=2 for PCA, and fit centroids on train before test prediction.

Hierarchical Clustering (Ward)

We generated a Ward dendrogram on a 5,000-row test sample for interpretive structure checks. We reused this same sample later for SCARF-side comparability.

DBSCAN with k-Distance Elbow

We tuned eps from 12-nearest-neighbor distance curves using KneeLocator maximum curvature. On a 50,000-row test sample we obtained PCA eps=26.2733, 17 clusters, and noise ratio 0.0023 (113 noise points).

6. Methodology - SCARF Contrastive Encoder

Architecture

We implemented SCARF from scratch in PyTorch. The encoder MLP was input → 256 → 128 → 64 with BatchNorm, ReLU, and Dropout(0.1). The projection head was 64 → 64 → 64. We trained on train split only, never using test rows during representation learning.

Corruption Strategy

We used corruption rate 0.6 at logical-feature level. For each sampled logical feature, we swapped values from a randomly selected donor row. We treated one-hot groups as atomic units so each corrupted categorical block remained valid.

def __getitem__(self, idx):
    x = self.data[idx].clone()
    x_corrupt = x.clone()
    perm = torch.randperm(self.n_logical)[: self.n_corrupt]

    cols_to_corrupt, donors_for_cols = [], []
    for p in perm.tolist():
        donor = torch.randint(0, self.n_samples, (1,)).item()
        if p < len(self.numeric_indices):
            cols_to_corrupt.append(self.numeric_indices[p])
            donors_for_cols.append(donor)
        else:
            group_idx = p - len(self.numeric_indices)
            for c in self.cat_groups[group_idx]:
                cols_to_corrupt.append(c)
                donors_for_cols.append(donor)

    if cols_to_corrupt:
        col_idx = torch.tensor(cols_to_corrupt, dtype=torch.long)
        donor_idx = torch.tensor(donors_for_cols, dtype=torch.long)
        x_corrupt[col_idx] = self.data[donor_idx, col_idx]

    return x, x_corrupt

NT-Xent / InfoNCE Objective

We used temperature 0.5 and computed normalized similarities on projection outputs. Positive pairs were original and corrupted views of the same row; negatives were all other samples in the batch.

def forward(self, z_i, z_j):
    batch_size = z_i.shape[0]
    z_i = F.normalize(z_i, dim=1)
    z_j = F.normalize(z_j, dim=1)

    z = torch.cat([z_i, z_j], dim=0)
    sim = torch.matmul(z, z.T) / self.temperature

    eye_mask = torch.eye(2 * batch_size, device=sim.device, dtype=torch.bool)
    sim = sim.masked_fill(eye_mask, -1e9)

    pos = torch.cat([torch.diag(sim, batch_size), torch.diag(sim, -batch_size)], dim=0)
    denom = torch.logsumexp(sim, dim=1)
    loss = -pos + denom
    return loss.mean()

Training Details

  • Epochs: 30
  • Batch size: 512
  • Optimizer: Adam (lr=1e-3, weight_decay=1e-5)
  • Scheduler: cosine annealing across epochs
  • Device in execution: CPU

Loss decreased from 5.7723 at epoch 1 to 5.5900 by epoch 30, indicating stable optimization.

Embedding Extraction and Euclidean K-Means Justification

We extracted raw encoder outputs (not unit-normalized) and intentionally clustered with Euclidean K-Means. Our rationale was that NT-Xent applies normalization inside projection space, while encoder norms may still carry intensity information (for example, more extreme borrower profiles). We explicitly chose not to discard this magnitude signal.

@torch.no_grad()
def extract_embeddings(encoder, X, batch_size=2048):
    encoder.eval()
    dataset = TensorDataset(torch.tensor(X, dtype=torch.float32))
    loader = DataLoader(dataset, batch_size=batch_size, shuffle=False)

    all_emb = []
    for (batch,) in loader:
        emb = encoder(batch.to(device, non_blocking=True))
        # We keep raw encoder outputs (unnormalized) intentionally.
        all_emb.append(emb.cpu().numpy())
    return np.vstack(all_emb)

8. Assumptions & Limitations

We explicitly acknowledge the following assumptions and constraints in our pipeline and interpretation:

  1. We assumed IID sampling across borrower rows, even though real portfolios can include temporal drift and policy shifts.
  2. We assumed median/mode imputation was acceptable under approximately missing-at-random behavior.
  3. We assumed DAYS_EMPLOYED=365243 was a placeholder anomaly rather than a meaningful encoded state.
  4. We assumed corruption rate 0.6 was suitable without dedicated tuning.
  5. We assumed NT-Xent temperature 0.5 was reasonable without sweep.
  6. We assumed 30 epochs were enough for useful convergence under our architecture and hardware budget.
  7. We assumed 64-dimensional embeddings were expressive enough for segmentation.
  8. We assumed Euclidean distance on unnormalized encoder outputs was a valid clustering geometry choice.
  9. We assumed a 90% PCA variance threshold was an acceptable baseline convention, despite arbitrariness.
  10. We assumed Ward linkage was informative even though it implicitly favors compact, roughly spherical structure.
  11. We relied on one primary dataset and did not perform external-dataset validation.
  12. We did not run comprehensive SCARF hyperparameter tuning (architecture, depth, width, augmentation schedule).
  13. We used bootstrap silhouette with only n=10 subsamples, which is limited for high-precision uncertainty quantification.
  14. We evaluated random train/test stability but did not perform out-of-time split validation.
  15. We treated downstream linear probe AUC as a useful diagnostic but not the training objective.

9. Honest Discussion of Results

We are explicit about the mixed outcome. PCA won clearly on linear probe AUC. If our only goal were to keep default signal linearly separable for a logistic model, PCA would be the stronger choice in this experiment. We do not reinterpret that away.

At the same time, SCARF won decisively on segmentation geometry and risk stratification at most shared k values. Test silhouette improved from 0.1797 to 0.3855, and IV became stronger for k=4,6,8. We therefore interpret SCARF as producing latent structure that is better aligned with unsupervised cluster quality, even if it is less linearly convenient.

This tradeoff is coherent with objective design. SCARF optimizes neighborhood structure through contrastive invariance under corruption, not supervised separation of the default label. PCA maximizes explained variance in linear subspace, which often aligns better with linear probes. These are different optimization targets, so divergence in metrics is expected.

We believe SCARF AUC could improve with additional work: supervised fine-tuning of encoder, longer training, larger networks, alternative projection heads, and hybrid loss terms that blend contrastive and supervised objectives. We did not run those extensions in this project, so we keep this as future work rather than claiming performance we did not measure.

Key Utility Function for Risk Separation

def weighted_information_value(labels, target):
    eval_df = pd.DataFrame({"cluster": labels, "target": target})
    total_good = int((eval_df["target"] == 0).sum())
    total_bad = int((eval_df["target"] == 1).sum())
    eps = 1e-9

    iv = 0.0
    for cluster_id, grp in eval_df.groupby("cluster"):
        good_i = int((grp["target"] == 0).sum())
        bad_i = int((grp["target"] == 1).sum())
        pct_good = max(good_i / total_good, eps)
        pct_bad = max(bad_i / total_bad, eps)
        iv += (pct_good - pct_bad) * np.log(pct_good / pct_bad)

    return float(iv)

12. Conclusion

We contributed a full leakage-safe segmentation pipeline that compares a strong classical baseline against a custom SCARF implementation for large-scale credit tabular data. We handled critical data issues (sentinel anomalies, missingness signal retention, split-consistent encoding), built semantically valid corruption for one-hot groups, and evaluated with geometry and risk-stratification metrics on held-out data.

Our final conclusion is nuanced: SCARF produced geometrically superior embeddings for borrower segmentation and stronger multi-cluster risk separation, while PCA remained stronger for linear discriminative default prediction. We view this not as contradiction but as objective mismatch across methods. If our downstream task is segmentation quality and cluster interpretability, SCARF is compelling; if our immediate objective is linear probe performance, PCA is still a robust baseline.

For future work, we would run broader hyperparameter sweeps, evaluate out-of-time stability, test additional tabular SSL encoders, and add supervised fine-tuning heads to improve AUC without sacrificing cluster geometry.
